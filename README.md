# Doctor Type Bot

Telegram-бот — интерактивный курс «Основы веб-технологий» для графических дизайнеров. Дипломный проект (ВКР).

Студент проходит уроки прямо в чате: читает короткие шаги с картинками, отвечает на практические задания, может задать вопрос встроенному ИИ-ассистенту. Прогресс сохраняется — можно вернуться в любой момент.

## 1. Что это и для кого

Бот учит дизайнеров основам веба: как работает интернет и браузер, базовый HTML и CSS, основы UX/UI. Материал разбит на **модули → уроки → шаги**. После части уроков идёт практика. Есть ИИ-ассистент (Doctor Type.Ai), который отвечает на вопросы по теме.

Курс рассчитан на параллельное прохождение с учёбой: не заменяет занятия, а помогает разобраться заранее или повторить.

Точные версии — в файле `requirements.txt`.

## 3. Структура проекта

```
doctor_type/
├── main.py              ← точка входа: запускает бота и связывает все части
├── requirements.txt     ← список библиотек
├── .env                 ← СЕКРЕТЫ (токены, ключи) — НЕ попадает в архив
├── credentials.json     ← ключ Google Таблиц — НЕ попадает в архив
│
├── handlers/            ← «обработчики»: реакции на нажатия кнопок и сообщения
│   ├── menu.py          ← главное меню, прогресс, обратная связь, сброс
│   ├── course.py        ← показ шагов урока, переход вперёд/назад
│   ├── practice.py      ← практические задания (5 типов)
│   ├── ai.py            ← режим общения с ИИ-ассистентом
│   ├── _images.py       ← file_id картинок для экранов
│   └── _helpers.py      ← вспомогательные функции (например, safe_edit)
│
├── content/             ← ВЕСЬ УЧЕБНЫЙ КОНТЕНТ
│   ├── modules.py       ← оглавление курса: модули, уроки, число шагов
│   ├── md_loader.py     ← читает тексты уроков из .md-файлов
│   ├── README.md        ← подробный гайд по написанию уроков и практик
│   └── steps/           ← сами тексты
│       └── module_01/
│           ├── lesson_01.md           ← текст урока (шаги)
│           └── lesson_01.practice.md  ← практика урока
│
├── ai/                  ← ИИ-ассистент
│   ├── client.py        ← обращение к Claude
│   ├── prompts.py       ← системная инструкция для ИИ (как себя вести)
│   ├── history.py       ← память диалога (в оперативке)
│   ├── rate_limit.py    ← лимит вопросов в день на пользователя
│   └── knowledge/       ← база знаний по урокам, которую читает ИИ
│
└── storage/
    └── sheets.py        ← сохранение прогресса в Google Таблицы
```

Простое правило: **тексты уроков — в `content/steps/`, оглавление — в `content/modules.py`, логика поведения — в `handlers/`.**

## 4. Как всё устроено простым языком

Представим бота как кофейню. Студент — клиент, Telegram — витрина для заказов, Railway или другой сервер — кухня, где всё готовится, Google Таблицы — журнал заказов, Claude — приглашённый консультант.

**Long polling** — способ, которым бот получает сообщения. Бот сам постоянно спрашивает Telegram: «Есть что-то новое? Студент что-то нажал?» Telegram отвечает «да, вот» или «пока нет». Для небольшого бота этого достаточно.

**Обработчики (handlers)** — условные сотрудники, каждый отвечает за свой тип действия. Нажал «Дальше» → работает `course.py`; ответил на задание → `practice.py`; написал вопрос ИИ → `ai.py`. В `main.py` прописано, какое действие какому обработчику отдавать (по «шаблонам» текста кнопки).

**Хостинг и внешние сервисы.** Railway — арендованный компьютер, который держит бота включённым 24/7 (~$5/мес). Google Таблицы и Claude — внешние сервисы: бот обращается к ним по интернету, когда нужно сохранить прогресс или ответить на вопрос.

**Роль GitHub.** GitHub — не часть работающего бота, а хранилище кода. Когда мы отправляем изменения (`git push`), Railway это замечает и сам пересобирает бота. То есть GitHub → Railway — это канал **обновления**, а не работы. Студент с GitHub никак не пересекается.

**Пример пути «нажал кнопку → ответ»:**
1. Студент нажимает «Дальше» → Telegram шлёт боту сигнал `step_module_01_lesson_01_3`.
2. `main.py` видит по шаблону, что это шаг урока → отдаёт обработчику `handle_step` в `course.py`.
3. Обработчик сохраняет прогресс в Google Таблицы, берёт текст шага из `.md`-файла и отправляет студенту новое сообщение с картинкой и кнопками.

## 5. Запуск на своём компьютере

**Шаг 1. Установить Python 3.10+** (python.org).

**Шаг 2. Установить библиотеки** — в папке проекта:
```bash
pip install -r requirements.txt
```

**Шаг 3. Создать файл `.env`** в корне проекта:
```env
BOT_TOKEN=токен_бота_от_BotFather
ANTHROPIC_API_KEY=ключ_от_Anthropic
GOOGLE_CREDENTIALS=credentials.json
```

Где брать значения:
- `BOT_TOKEN` — у [@BotFather](https://t.me/BotFather) в Telegram (команда `/newbot` или `/token`).
- `ANTHROPIC_API_KEY` — в личном кабинете Anthropic (console.anthropic.com).
- `GOOGLE_CREDENTIALS` — путь к файлу `credentials.json` (локально) или сам JSON-текст (на сервере).

**Шаг 4. Положить `credentials.json`** в корень — это ключ доступа к Google Таблицам (service account из Google Cloud). У сервисного аккаунта должен быть доступ к нужной таблице.

**Шаг 5. Запустить:**
```bash
python main.py
```
Появится «Бот запущен» — можно писать боту в Telegram. Остановить — `Ctrl+C`.

## 6. Деплой (публикация в интернет)

Бот работает на **Railway** и обновляется автоматически из **GitHub**.

**Как выглядит обновление:**
```
меняешь код → git push → GitHub → Railway сам пересобирает и перезапускает бота
```
То есть чтобы выкатить изменения, достаточно отправить их в GitHub:
```bash
git add .
git commit -m "описание изменений"
git push
```
Репозиторий: `https://github.com/YaroslavMyagkov/doctor-type-bot`.

**Секреты на сервере.** На Railway нет файлов `.env` и `credentials.json` — вместо них в дашборде Railway (раздел **Variables**) заданы переменные: `BOT_TOKEN`, `ANTHROPIC_API_KEY` и `GOOGLE_CREDENTIALS` (сюда вставлено всё содержимое `credentials.json` как текст).

**Команда запуска.** В проекте нет файла `Procfile`, поэтому команда запуска задаётся в дашборде Railway: **Settings → Deploy → Start Command → `python main.py`**. Если понадобится сменить способ запуска — менять нужно там.

## 7. Как добавить урок, ничего не сломав

Тексты уроков лежат в `content/steps/`, а оглавление — в `content/modules.py`. Их нужно держать согласованными.

**Главное правило (чтобы не сломать бота):**
> Сколько `##` заголовков в файле урока — столько пустых `{}` должно быть в списке `steps` в `content/modules.py`.
> Каждый `##` = один шаг = одно сообщение в боте.

**Порядок действий:**

1. **Создать файл урока** `content/steps/module_XX/lesson_YY.md`. Каждый шаг начинается с `##`:
   ```markdown
   ## Заголовок первого шага
   image: file_id_картинки   (строка необязательна)

   Текст шага. **Жирный** и *курсив* работают.

   ## Заголовок второго шага

   Текст второго шага.
   ```

2. **Прописать урок в `content/modules.py`** — в нужном модуле, в списке `lessons`:
   ```python
   {
       "id": "lesson_YY",
       "title": "Урок N. Название",
       "steps": [{}, {}],   # ← два {}, потому что два ## в файле
       "practice": []
   }
   ```

3. **Практика (по желанию)** — файл `content/steps/module_XX/lesson_YY.practice.md`. Задания разделяются строкой `---`, каждое начинается с типа. Пять типов: `# choice`, `# true_false`, `# open`, `# case`, `# photo`. Точный синтаксис каждого — в **`content/README.md`** (там уже есть готовый гайд с примерами).

4. **База знаний для ИИ (по желанию)** — чтобы ассистент отвечал по новому уроку, добавь `ai/knowledge/module_XX_lesson_YY.md` и пропиши связь в `ai/knowledge/_index.py`.

5. **Картинки.** Картинки хранятся в Telegram и подключаются по `file_id`. Чтобы его получить: отправь боту фото — он ответит `file_id`. Скопируй и вставь в `.md` (строка `image:`) или в `modules.py` (`practice_image`, `finish_image`).

**Чеклист перед публикацией:**
- [ ] Число `##` в `.md` = числу `{}` в `steps`.
- [ ] Задания в практике разделены строкой `---` (ровно три дефиса).
- [ ] У `# case` есть все `outcome_A…D` и `best`; у `# photo` есть `file_id`.

## 8. Как перевести бота в сайт

Если Telegram неудобен, курс можно превратить в **простой статический сайт** (обычные HTML/CSS/JS, без сложных фреймворков) и разместить бесплатно на GitHub Pages.

**Что переносится почти без изменений:**
- Учебный контент — те же `.md`-файлы из `content/steps/`.
- Пять типов практик — та же логика (выбор варианта, верно/неверно, открытый вопрос, кейс, фото-вопрос).

**Что заменить:**
- Прогресс студента — вместо Google Таблиц хранить в браузере (`localStorage`). Не нужен сервер.
- ИИ-ассистент — на первом этапе можно убрать (он требует секретного ключа, который нельзя публиковать в статическом сайте).

**Целевая структура сайта:**
```
site/
├── index.html        ← оглавление курса
├── lesson.html       ← страница урока (шаги + практика)
├── css/style.css
├── js/app.js         ← навигация, проверка ответов, прогресс в localStorage
└── content/          ← тексты уроков (те же .md или переведённые в JSON)
```

**Готовый план-промпт для Claude Code** (открой Claude Code в папке проекта и попроси примерно так):
> «Есть учебный курс в формате Telegram-бота. Контент уроков лежит в `content/steps/` (.md-файлы, шаги разделены `##`), практики — в `.practice.md` (5 типов, разделены `---`). Оглавление — в `content/modules.py`. Сделай простой статический сайт на HTML/CSS/JS: главная с оглавлением, страницы уроков с пошаговым просмотром и практикой, прогресс сохраняй в localStorage. ИИ-ассистент пока не переноси. Опиши, как разместить результат на GitHub Pages.»

Claude Code прочитает контент и соберёт сайт по этой структуре.

## 9. Где хранятся картинки и макеты (Figma)

Все иллюстрации для экранов бота нарисованы в Figma:
👉 [Doctor Type — макет в Figma](https://www.figma.com/design/V3mpPbTrgikNFEI2twjwG6/Доктор-Тайп?node-id=0-1&t=Tfr77WWB0gRBAqiV-1)

**Важно:** доступ к файлу — только «может копировать» (view/copy), редактировать оригинал напрямую нельзя. Перед тем как что-то менять, сделай себе копию — в Figma это **«Copy to your drafts»** («Скопировать в черновики») — и редактируй уже свою копию, а не общий файл.

**Как добавить картинку в бота (это то, как устроены все текущие изображения в проекте):**
1. Экспортируй картинку из Figma в PNG или JPG.
2. Отправь её боту в Telegram обычным сообщением (как фото).
3. Бот пришлёт в ответ `file_id` — длинную строку вида `AgACAgIAAxkBAAIB...`.
4. Скопируй `file_id` и вставь в нужное место:
   - для картинки шага урока — строка `image: <file_id>` под заголовком шага в `.md`-файле;
   - для общих экранов (меню, ИИ, фидбэк и т.п.) — в `handlers/_images.py`;
   - для финала урока / практики — поля `practice_image` / `finish_image` в `content/modules.py`.

Подробнее про вставку картинок в урок — раздел 7, пункт 5.
