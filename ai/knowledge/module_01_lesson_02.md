## Quick reference

- Браузер строит страницу последовательно: сначала HTML (скелет), потом CSS (стили), потом изображения, шрифты, скрипты
- HTML без CSS — это страница как в 1996 году: просто текст, никаких стилей
- Чем тяжелее файл — тем позже загружается; изображения грузятся последними
- Белый экран при загрузке = браузер ждёт тяжёлый файл (чаще JavaScript), без которого не начинает отрисовку
- DOM (Document Object Model) — представление HTML-документа в памяти браузера в виде дерева объектов
- HTML — текстовый файл с разметкой. DOM — живое дерево в памяти браузера. Это не одно и то же
- 53% пользователей закрывают сайт, если он не загрузился за 3 секунды (по данным Google)
- Каждый подключённый шрифт, тяжёлое изображение, лишний скрипт — это пауза перед тем, как пользователь увидит страницу
- Кэш — браузер сохраняет уже загруженные файлы, при повторном визите берёт их из памяти
- Очистить кэш — способ «перезагрузить» сайт с нуля, если он отображается неправильно

**Форматы изображений:**
- JPG — для фотографий, хорошо сжимает без большой потери качества
- PNG — для иконок и графики с прозрачностью, где важна пиксельная точность
- WebP — современный формат, лучше JPG и PNG; сборщики (Vite, Next.js) конвертируют в него автоматически
- SVG — векторный формат в виде кода. Масштабируется без потери качества, анимируется через CSS, цвет меняется в коде. Стандарт для иконок и логотипов в вебе
- AI и EPS — форматы иллюстраторских файлов, в веб напрямую не идут

**Шрифты в вебе:**
- WOFF2 — стандартный шрифтовой формат для веба, сжатый и оптимизированный
- Пока шрифт не загрузился: FOUT = показывает системный шрифт-заглушку, FOIT = текст невидим до загрузки
- Системные шрифты (Arial, Georgia, San Francisco на Apple) не нужно грузить — они уже есть на устройстве
- Google Fonts — бесплатная библиотека шрифтов, подключается одной строкой
- Если подключаешь шрифт с 30 начертаниями, но используешь 5 — подключи только 5
- В вебе нет фиксированной ширины страницы, текст перетекает. Переносы браузер расставляет сам
- Неразрывный пробел (& n b s p ;) — ставится перед предлогом, чтобы он не висел в конце строки
- Пользователь может изменить размер шрифта в браузере; хороший дизайн выдерживает это

**DevTools (инструменты разработчика):**
- Открыть DevTools: F12 (или правая кнопка → «Исследовать элемент»)
- Вкладка Elements — HTML-структура страницы и CSS-стили каждого элемента; наводишь на элемент — видишь отступы, шрифты, цвета
- Вкладка Network — все файлы, которые браузер запросил: их вес, время загрузки, что взял из кэша
- Куки — небольшие файлы на устройстве пользователя, которые сайт сохраняет для «запоминания» пользователя
- Режим инкогнито не сохраняет куки между сессиями, но провайдер по-прежнему видит трафик

## Anti-patterns

Не путай DOM с HTML. HTML — это файл с текстовой разметкой. DOM — дерево объектов в памяти браузера, с которым работает JavaScript. Одно — файл, другое — структура в памяти.

Не говори «рендеринг» без объяснения — это слово впервые встречается именно в этом уроке. Рендеринг = процесс отрисовки браузером готовой страницы из полученных файлов.

Не путай WebP с чем-то, что нужно делать вручную. Сборщики сами конвертируют JPG/PNG в WebP. Дизайнеру не нужно экспортировать в WebP.

Не говори, что SVG — только для иконок. SVG подходит для любой векторной графики: иллюстрации, диаграммы, анимированные элементы. Ограничение: плохо подходит для сложных детализированных изображений (фото, тени).

Не путай хорошее использование DevTools с «взломом» сайта. HTML и CSS любого сайта открыты — это не секретные данные, это нормальная часть работы дизайнера и разработчика.

## Подробно

**Почему сайт иногда загружается странно — сначала текст без стилей?**
Браузер загружает файлы последовательно. HTML приходит первым — и страница уже существует как скелет без оформления. CSS добавляет стили. Изображения грузятся последними, потому что они тяжелее. Чем медленнее интернет, тем заметнее этот процесс.

**Что такое DOM и зачем он нужен?**
DOM (Document Object Model) — это представление HTML-страницы в памяти браузера в виде дерева объектов. Браузер читает HTML-файл и строит из него DOM. JavaScript работает именно с DOM — добавляет, удаляет, меняет элементы на лету, без перезагрузки страницы.

**Какой формат изображений использовать в вебе?**
Фотографии — JPG или WebP. Графика с прозрачностью (иконки, баннеры с прозрачным фоном) — PNG. Иконки, логотипы, простые иллюстрации — SVG, потому что он масштабируется без потери качества и весит мало. WebP в целом лучший выбор, но его генерирует сборщик, не дизайнер.

**Почему SVG лучше PNG для иконок?**
SVG хранится как код, а не как пиксели. Поэтому: масштабируется на любой экран без размытия, цвет меняется в CSS-коде без замены файла, можно анимировать, файл весит меньше. Минус: не подходит для фотографий и сложных детализированных изображений.

**Как открыть DevTools и что там смотреть?**
Нажми F12 в любом браузере. Вкладка Elements — наводишь на любой элемент страницы и видишь его HTML-структуру, точные размеры, отступы, шрифт, цвет. Полезно проверить, правильно ли разработчик реализовал макет. Вкладка Network — все файлы, которые браузер запрашивал: что тяжёлое, что долго грузилось, что взяли из кэша.

**Что такое куки и зачем они нужны?**
Куки — небольшие файлы на твоём устройстве, которые сайт сохраняет, чтобы «запомнить» тебя. Поэтому маркетплейс показывает твои недавние товары без логина — он узнаёт тебя по куки. Поэтому везде баннеры «принять куки» — сайт спрашивает разрешения. В режиме инкогнито куки не сохраняются, но провайдер всё равно видит трафик.

**Как очистить кэш и зачем?**
Кэш — файлы, которые браузер сохраняет при первом посещении (картинки, шрифты, скрипты), чтобы не скачивать их снова. Если сайт обновился, но ты видишь старую версию — очисти кэш. Браузер загрузит всё заново.

**Почему важно подключать только нужные начертания шрифта?**
Каждое начертание — отдельный файл, который браузер скачивает с сервера. Если подключить 30 начертаний и использовать 5 — браузер всё равно сделает 30 запросов. Это замедляет загрузку страницы без всякой пользы.
