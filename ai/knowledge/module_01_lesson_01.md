## Quick reference

- Интернет — сеть связанных компьютеров. Одни — клиенты (браузеры), другие — серверы (хранят данные)
- Сервер — такой же компьютер, просто с другим железом под другие задачи
- Дата-центры — здания с тысячами серверов, где хранится большинство сайтов
- IP-адрес — уникальный числовой адрес каждого устройства в интернете, например 142.250.74.46
- Домен — человекочитаемое имя сайта (google.com), DNS переводит его в IP
- DNS — служба, которая переводит домен в IP-адрес за миллисекунды
- Хостинг — аренда места на сервере для хранения файлов сайта
- Tilda хранит файлы на своих серверах, поэтому адрес вида username.tilda.ws — это их поддомен
- Для своего сайта нужно купить два сервиса: хостинг и домен
- HTTP/HTTPS — протоколы, по которым клиент и сервер общаются
- HTTPS = зашифрованная версия HTTP, замочек в адресной строке
- Сайты без HTTPS браузеры помечают как небезопасные
- Физически данные идут по кабелям. Wi-Fi в какой-то момент тоже уходит в кабель
- 95% мирового интернет-трафика идёт по подводным оптоволоконным кабелям, не через спутники
- Cloudflare и аналоги — посредники: защищают от DDoS-атак, ускоряют загрузку через кэш ближайшего к пользователю узла
- VPN — добавляет ещё один сервер-посредник, шифрует данные, скрывает настоящее местоположение
- Прокси — то же, что VPN, но без шифрования: быстрее, менее безопасно
- Сайт = HTML (структура) + CSS (оформление) + JavaScript (интерактивность)
- HTML — скелет: текст, ссылки, изображения, иерархия блоков
- CSS — визуальное оформление: цвета, шрифты, отступы, позиционирование
- JavaScript — интерактивность: анимации, слайдеры, реакция на клик
- Figma, Illustrator, Photoshop не создают рабочий код — только макет
- Между макетом и сайтом всегда стоит разработчик, который переводит дизайн в HTML/CSS/JS вручную
- Не всё, что легко нарисовать в Figma, легко реализовать в браузере
- «Обсуждать ограничения с разработчиком до того, как макет готов» — правильный подход
- На собеседованиях в продуктовые команды спрашивают: «Что делать, если разработчик не может реализовать дизайн?»

## Anti-patterns

Не путай домен и хостинг. Домен — это адрес (имя). Хостинг — место хранения файлов. Это две разные услуги.

Не говори, что Figma создаёт код или сайт. Figma создаёт макет — визуальное представление, которое разработчик переводит в код вручную (или с помощью ИИ-инструментов).

Не путай HTML и весь сайт. HTML — только один из трёх слоёв. Без CSS страница выглядит как документ 1996 года, без JavaScript — нет интерактивности.

Не говори, что сервер — это что-то сложное и недоступное. Технически любой компьютер может быть сервером, вопрос только в производительности.

## Подробно

**Что такое клиент-серверная архитектура?**
Интернет устроен так, что одни устройства хранят данные (серверы), другие их запрашивают (клиенты, то есть мы с нашими браузерами). Когда ты открываешь сайт, браузер отправляет запрос на сервер, сервер отвечает файлами, браузер собирает из них страницу.

**Что такое DNS и зачем он нужен?**
DNS — это как телефонная книга интернета. Ты вводишь google.com, DNS переводит это в IP-адрес 142.250.74.46 и направляет запрос на нужный сервер. Происходит за миллисекунды, незаметно.

**Как создать свой сайт?**
Нужно две вещи: хостинг (арендовать место на сервере для файлов) и домен (купить адрес сайта). Домен привязывается к хостингу, и сайт становится доступен по нужному адресу. Если делать сайт в Tilda — про это думать не нужно, Tilda предоставляет своё хранение и поддомен.

**Зачем нужен HTTPS?**
HTTPS шифрует данные между браузером и сервером. Это особенно важно при вводе паролей, данных карт. Браузеры помечают HTTP-сайты как небезопасные, поэтому HTTPS — стандарт.

**Что такое VPN и как он работает?**
VPN добавляет промежуточный сервер в другой стране. Провайдер видит подключение к этому серверу, а не к конечному сайту. Сайт видит IP другой страны, а не твой. Данные шифруются. Прокси работает похоже, но без шифрования.

**Как макет попадает на сайт?**
Никак сам по себе. Figma — инструмент для дизайна, а не для создания сайтов. Разработчик смотрит на макет и переводит его в HTML, CSS и JavaScript. Изображения и иконки экспортируются из Figma и подключаются в код.

**Почему разработчик не всегда делает точно по макету?**
Потому что браузерная среда накладывает ограничения. Не всё, что рисуется в Figma, одинаково просто реализовать в коде. Профессиональный дизайнер обсуждает технические ограничения с разработчиком ещё до финального макета.

**Что такое Cloudflare и зачем он нужен?**
Cloudflare — компания-посредник. Она кэширует данные популярных сайтов на серверах по всему миру — так страница грузится с ближайшего к тебе узла, а не с сервера в другой стране. Ещё Cloudflare защищает от DDoS-атак (когда сервер забивают тысячами фейковых запросов).
