## Гайд по созданию контента уроков

Все файлы лежат в `content/steps/`. Структура:
```
content/steps/
  module_01/
    lesson_03.md              ← текст урока
    lesson_03.practice.md     ← практика (опционально)
```

---

## Файл урока `lesson_XX.md`

Каждый `#`-заголовок = новый шаг (одно сообщение в боте). Уровень не важен — хоть `#`, хоть `###`.

```markdown
## Заголовок первого шага

Обычный текст абзаца. Можно несколько абзацев — просто оставляй пустую строку между ними.

Второй абзац того же шага.

**Жирный текст** и *курсив* работают как обычно.

- Список через дефис
- Второй пункт

## Заголовок второго шага

Текст второго шага.
```

**Что куда превращается в Telegram:**

| Markdown | Telegram |
|---|---|
| `**Жирный**` | **Жирный** |
| `*Курсив*` | _Курсив_ |
| `- пункт` | — пункт |
| `## Заголовок` | **Заголовок** (первая строка шага) |

> ⚠️ Важно: сколько `##` заголовков в файле — столько `{}` должно быть в `modules.py` в списке `steps`. Добавил шаг — добавь `{}`.

## Файл практики `lesson_XX.practice.md`

Задания разделяются строкой `---`. Каждое начинается с `# тип`.

### Тест с вариантами — `# choice`

```markdown
# choice

Что такое viewport?

A: Физический размер экрана
B: Видимая область страницы в окне браузера
C: Единица измерения в CSS
D: Тип HTML-тега

correct: B
ok: Верно. Viewport — видимая область, именно по ней строится адаптивная вёрстка.
fail: Не то. Viewport — это видимая область страницы, а не физический размер экрана.
```

---

### Верно/неверно — `# true_false`

```markdown
# true_false

JPG поддерживает прозрачность.

correct: false
ok: Верно. JPG не поддерживает прозрачность — для этого нужен PNG или SVG.
fail: Не так. JPG прозрачность не поддерживает — только PNG и SVG.
```

---

### Открытый вопрос — `# open`

```markdown
# open

Ты делаешь лендинг и хочешь добавить логотип. Какой формат выберешь и почему?

sample: SVG — логотип векторный, масштабируется без потерь и легко меняет цвет через CSS. PNG только если SVG недоступен.
```

### Кейс — `# case`

```markdown
# case

situation: Клиент просит добавить на сайт 5 красивых шрифтов. Каждый — отдельный файл, итого 15 файлов.
question: Что скажешь клиенту?

A: Добавлю все пять, клиент всегда прав
B: Объясню, что много шрифтов замедляют загрузку, и предложу 1–2
C: Добавлю один шрифт без объяснений
D: Откажусь от задачи

outcome_A: Сайт грузится медленно. Клиент доволен внешним видом, но пользователи уходят.
outcome_B: Клиент понял проблему. Сайт быстрый и выглядит профессионально.
outcome_C: Клиент удивлён — он хотел другое. Доверие потеряно.
outcome_D: Потерял проект без объяснений.

best: B
note: Дизайнер помогает клиенту принять правильное решение, а не просто выполняет задание.
```

### Фото-тест — `# photo`

```markdown
# photo

file_id: AgACAgIAAxkB...  ← file_id из Telegram (отправь боту фото командой /start и скопируй)

На скриншоте показана вкладка DevTools. Как она называется?

A: Console
B: Elements
C: Sources
D: Network

correct: D
ok: Верно. Network показывает все файлы, загруженные браузером.
fail: Не то. Это вкладка Network — видны запросы и вес файлов.
```
